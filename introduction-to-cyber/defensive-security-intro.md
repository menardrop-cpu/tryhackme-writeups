# Defensive Security Intro

**Platform** : TryHackMe  
**Difficulty** : Easy  
**Time** : 15 min  
**Status** : Completed 100%

## Objective

Understand what defensive security is, what it covers in the real world,
and get hands-on with a practical defence scenario.

## Tasks covered

* Task 1 : Introduction to Defensive Security
* Task 2 : Responsibilities of Defensive Security
* Task 3 : Defensive Security in Practice
* Task 4 : Practical : Defend FakeBank

## What I did

The room introduces the defensive security mindset and the concept of
Defence in Depth : layering multiple protections so that if one fails,
others remain standing.

The practical task involved defending FakeBank against an active attack,
applying the concepts seen throughout the room in a simulated environment.

## Key concepts

* Defence in Depth : employee training, IDS, firewalls, security policies
* SOC (Security Operations Centre) : monitors alerts, investigates anomalies,
responds to incidents around the clock
* SIEM : central system aggregating data from all security devices to give
visibility across the organisation

## Takeaway

Defensive security is not passive. It is a structured, layered approach
where humans, processes, and tools work together. The SOC is the operational
core of this defence, and SIEM is its nervous system. Understanding both is
essential for any GRC role where you need to assess whether an organisation
has adequate controls in place.
